# 개인정보 보호 정책

최종 업데이트: 2026-06-14

이 개인정보 보호 정책은 `discord_search_bot` 프로젝트의 공개 초대 링크로 사용할 수 있는 데모 Discord 봇 인스턴스에 적용됩니다.

이 봇은 상시 켜져 있을 수 있지만, 주 목적은 기능을 시험해 볼 수 있는 데모 제공입니다. 안정적인 사용, 개인정보 통제, 데이터 보관 관리를 위해서는 사용자가 각자 자신의 Discord 애플리케이션을 만들고 직접 봇을 운영하는 것을 권장합니다.

데모 봇은 예고 없이 기능이 제한되거나 중단될 수 있으며, 언제든지 종료될 수 있습니다. 직접 봇을 운영하는 경우, 해당 인스턴스의 데이터 처리와 개인정보 보호 책임은 그 운영자에게 있습니다.

이 문서는 법률 자문이 아닙니다. 직접 봇을 운영하는 경우에는 실제 운영 방식, 관할 법률, 연락처에 맞게 별도의 문서를 검토해야 합니다.

## 수집하는 데이터

데모 봇은 기능 제공을 위해 다음 데이터를 처리할 수 있습니다.

- Discord 사용자 ID, 사용자명 또는 표시 이름
- 서버, 채널, 메시지 ID
- 메시지 작성 시각
- 검색 대상 메시지의 본문
- `/notify_version` 설정에 필요한 사용자 ID, 알림 설정, 마지막 알림 버전
- 채널별 캐싱 설정
- 명령어 등록, 오류 분석, 운영 안정성을 위한 로그

데모 봇은 광고, 사용자 프로파일링, 데이터 판매, AI 학습을 목적으로 데이터를 수집하지 않습니다.

## 메시지 본문 처리

이 봇의 주요 기능은 Discord 기본 검색이 잘 찾지 못하는 한글 메시지를 검색하는 것입니다. 데모 봇도 이 기능을 제공하기 위해 메시지 본문에 접근합니다.

캐싱이 꺼진 채널에서는 `/search` 명령 실행 중 Discord API로 과거 메시지를 임시로 읽고, 검색어와 일치하는 결과를 명령 실행자에게 DM으로 전송합니다. 이 경우 검색 대상 메시지를 영구 저장하지 않습니다.

캐싱이 켜진 채널에서는 서버 관리 권한을 가진 사용자가 `/config caching true`를 실행한 뒤부터 해당 채널의 메시지 본문과 관련 메타데이터를 데이터베이스에 저장할 수 있습니다. 캐싱은 채널 단위로 동작하며, 검색 속도를 높이기 위한 목적으로만 사용됩니다. 캐싱이 켜지기 이전의 데이터도 캐싱할 수 있습니다.

검색 결과 DM에는 메시지 작성자, 작성 시각, 메시지 링크, 메시지 일부 내용이 포함될 수 있습니다.

## 로그

데모 봇은 운영 상태 확인과 오류 분석을 위해 로그를 남깁니다. 로그에는 사용자 메시지 본문, 검색어, 검색 결과 내용을 기록하지 않습니다.

로그에는 다음 정보가 포함될 수 있습니다.

- 봇 시작, 데이터베이스 초기화, 명령어 등록 같은 운영 상태
- 명령어 등록이 수행된 Discord 서버 이름과 서버 ID
- 캐시 처리, 버전 확인, DM 알림 처리 중 발생한 오류 메시지
- 오류가 발생한 경우 관련 Discord 사용자 ID 또는 채널/서버 ID

로그는 문제 해결과 운영 안정성을 위해서만 사용됩니다.

## 데이터 저장 및 보관

저장 데이터는 데모 봇 운영 환경의 SQLite 데이터베이스에 보관됩니다. 데모 봇의 경우 데이터베이스를 암호화 합니다.

SQLCipher 키가 설정된 환경에서는 암호화된 SQLite 데이터베이스를 사용합니다. 키가 설정되지 않은 환경에서는 평문 SQLite 데이터베이스를 사용합니다.

캐싱된 채널 메시지는 다음 중 하나가 발생할 때까지 보관될 수 있습니다.

- 서버 관리자가 `/config caching false`로 해당 채널의 캐싱을 끈 경우
- Discord에서 원본 메시지 삭제 이벤트를 봇이 수신한 경우
- 사용자가 삭제를 요청하고 운영자가 해당 요청을 처리한 경우
- 데모 봇 운영 종료, 법적 요구, Discord 정책 요구 등으로 삭제가 필요한 경우

데모 봇이 오프라인인 동안 발생한 메시지 삭제나 권한 변경은 자동으로 반영되지 않을 수 있습니다. 이 경우 삭제 요청을 통해 수동 처리를 요청할 수 있습니다.

## 데이터 공유

운영자는 저장 데이터를 판매하거나 광고 목적으로 공유하지 않습니다.

다음 경우에는 데이터가 제3자에게 처리되거나 제공될 수 있습니다.

- Discord API와 Discord 서비스가 봇 기능 제공을 위해 데이터를 처리하는 경우
- 봇 운영에 필요한 서버, 호스팅, 저장소, 로그 시스템이 데이터를 처리하는 경우
- 법률, 수사기관, 법원 명령, Discord 정책 또는 보안상 필요한 요청에 응답해야 하는 경우

## 사용자 및 서버 관리자의 선택권

서버 관리자는 `/config caching true`로 채널 캐싱을 켤 수 있고, `/config caching false`로 끌 수 있습니다. 캐싱을 끄면 해당 채널에 저장된 메시지 데이터가 삭제됩니다.

사용자는 자신의 데이터 삭제를 요청할 수 있습니다. 삭제 요청에는 가능한 경우 Discord 사용자 ID, 서버 ID, 채널 ID, 관련 메시지 링크를 포함해 주세요. 요청은 GitHub Issues를 통해 보낼 수 있습니다.

GitHub Issues는 공개될 수 있으므로 봇 토큰, 민감한 개인정보, 메시지 원문은 올리지 마세요. 공개하기 곤란한 정보가 필요한 경우 운영자가 별도 연락 방법을 안내할 수 있습니다. 공개 운영에 사용하기 전에는 이메일 또는 지원 서버 같은 비공개 문의 창구로 교체하는 것을 권장합니다.

문의 및 삭제 요청: <https://github.com/80ROkWOC4j/discord_search_bot/issues>

## 직접 운영하는 경우

이 저장소의 코드를 직접 실행하는 운영자는 자신의 봇 인스턴스에서 수집, 저장, 공유되는 데이터에 대해 책임을 집니다. 직접 운영자는 자신에게 맞는 개인정보 보호 정책과 이용 약관을 작성하고, Discord Developer Portal에 해당 링크를 등록해야 합니다.

직접 운영자는 다음 사항을 검토해야 합니다.

- 데이터베이스 암호화 키 설정 여부
- 데이터 백업 및 로그 보관 여부
- 삭제 요청 처리 방법
- 봇을 초대한 서버 구성원에게 캐싱 사실을 알리는 방법
- 적용되는 지역 개인정보 보호 법률

## 보안

데모 봇은 데이터 접근을 기능 제공에 필요한 범위로 제한하려고 합니다. 다만 어떤 시스템도 완전한 보안을 보장할 수 없습니다. 직접 운영자는 봇 토큰, 데이터베이스 파일, SQLCipher 키, 서버 접근 권한을 안전하게 관리해야 합니다.

## 변경

이 정책은 봇 기능, 운영 방식, Discord 정책 변경에 따라 업데이트될 수 있습니다. 중요한 변경 사항은 저장소 문서 업데이트 또는 릴리스 노트를 통해 알릴 수 있습니다.
